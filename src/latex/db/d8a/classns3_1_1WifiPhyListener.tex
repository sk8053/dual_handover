{\em duration} & the expected busy duration.\\

This method does not really report a real state change as opposed to the other methods in this class. It merely reports that, unless the medium is reported busy through Notify\+Tx\+Start or Notify\+Rx\+Start/\+End, it will be busy as defined by the currently selected C\+CA mode.

Typical client code which wants to have a clear picture of the C\+CA state will need to keep track of the time at which the last Notify\+Cca\+Busy\+Start method is called and what duration it reported.
