\subsection{Detailed Description}
The abstract base class of a Frequency Reuse algorithm. 

Generally Frequency reuse algorithm tells the \hyperlink{classns3_1_1Scheduler}{Scheduler} which RB can be allocated and which can not. FR policy depend on its implementation.

The communication with the e\+NodeB M\+AC \hyperlink{classns3_1_1Scheduler}{Scheduler} instance is done through the {\itshape Lte\+Ffr\+Sap} interface. The frequency reuse algorithm instance corresponds to the \char`\"{}provider\char`\"{} part of this interface, while the e\+NodeB M\+AC \hyperlink{classns3_1_1Scheduler}{Scheduler} instance takes the role of the \char`\"{}user\char`\"{} part.

The communication with the e\+NodeB R\+RC instance is done through the {\itshape Lte\+Ffr\+Rrc\+Sap} interface. The frequency reuse algorithm instance corresponds to the \char`\"{}provider\char`\"{} part of this interface, while the e\+NodeB R\+RC instance takes the role of the \char`\"{}user\char`\"{} part.
