\subsection{Detailed Description}
Logging functions and macros. 

L\+OG functionality\+: macros which allow developers to send information to the {\ttfamily std\+::clog} output stream.

All logging messages are disabled by default. To enable selected logging messages, use the \hyperlink{namespacens3_adc4ef4f00bb2f5f4edae67fc3bc27f20}{ns3\+::\+Log\+Component\+Enable} function or use the N\+S\+\_\+\+L\+OG environment variable

Use the environment variable N\+S\+\_\+\+L\+OG to define a \textquotesingle{}\+:\textquotesingle{}-\/separated list of logging components to enable. For example (using bash syntax), 
\begin{DoxyCode}
$ \hyperlink{log-macros-disabled_8h_ab01b4392d98b06fb128e77c85129a477}{NS\_LOG}=\textcolor{stringliteral}{"OlsrAgent"} ./waf --\hyperlink{generate__test__data__lte__spectrum__value__txpsd_8m_a093eaf8ac95b223aafb28e4668160d1f}{run} ...
\end{DoxyCode}
 would enable one component at all log levels. 
\begin{DoxyCode}
$NS\_LOG=\textcolor{stringliteral}{"OlsrAgent:Ipv4L3Protocol"} ./waf --\hyperlink{generate__test__data__lte__spectrum__value__txpsd_8m_a093eaf8ac95b223aafb28e4668160d1f}{run} ...
\end{DoxyCode}
 would enable two components, at all log levels, etc. {\ttfamily N\+S\+\_\+\+L\+OG=\char`\"{}$\ast$\char`\"{}} will enable all available log components at all levels.

To control more selectively the log levels for each component, use this syntax\+: 
\begin{DoxyCode}
$ \hyperlink{log-macros-disabled_8h_ab01b4392d98b06fb128e77c85129a477}{NS\_LOG}=\textcolor{stringliteral}{'Component1=func|warn:Component2=error|debug'}
\end{DoxyCode}
 This example would enable the {\ttfamily func}, and {\ttfamily warn} log levels for \textquotesingle{}Component1\textquotesingle{} and the {\ttfamily error} and {\ttfamily debug} log levels for \textquotesingle{}Component2\textquotesingle{}. The wildcard \textquotesingle{}$\ast$\textquotesingle{} can be used here as well. For example {\ttfamily N\+S\+\_\+\+L\+OG=\textquotesingle{}$\ast$=level\+\_\+all$\vert$prefix\textquotesingle{}} would enable all log levels and prefix all prints with the component and function names.

A note on \hyperlink{log-macros-disabled_8h_a90b90d5bad1f39cb1b64923ea94c0761}{N\+S\+\_\+\+L\+O\+G\+\_\+\+F\+U\+N\+C\+T\+I\+O\+N()} and \hyperlink{log-macros-disabled_8h_a8f7e4afc291c9d29a65c18ac1f79197b}{N\+S\+\_\+\+L\+O\+G\+\_\+\+F\+U\+N\+C\+T\+I\+O\+N\+\_\+\+N\+O\+A\+R\+G\+S()}\+: generally, use of (at least) \hyperlink{log-macros-disabled_8h_a90b90d5bad1f39cb1b64923ea94c0761}{N\+S\+\_\+\+L\+O\+G\+\_\+\+F\+U\+N\+C\+T\+I\+O\+N(this)} is preferred, with the any function parameters added\+: 
\begin{DoxyCode}
\hyperlink{log-macros-disabled_8h_a90b90d5bad1f39cb1b64923ea94c0761}{NS\_LOG\_FUNCTION} (\textcolor{keyword}{this} << arg1 << args);
\end{DoxyCode}
 Use \hyperlink{log-macros-disabled_8h_a8f7e4afc291c9d29a65c18ac1f79197b}{N\+S\+\_\+\+L\+O\+G\+\_\+\+F\+U\+N\+C\+T\+I\+O\+N\+\_\+\+N\+O\+A\+R\+G\+S()} only in static functions with no arguments.

Define a Log component with a specific name.

This macro should be used at the top of every file in which you want to use the N\+S\+\_\+\+L\+OG macro. This macro defines a new \char`\"{}log component\char`\"{} which can be later selectively enabled or disabled with the \hyperlink{namespacens3_adc4ef4f00bb2f5f4edae67fc3bc27f20}{ns3\+::\+Log\+Component\+Enable} and \hyperlink{namespacens3_ad9b33adcd3aa5fc02ae8e8a627d138d5}{ns3\+::\+Log\+Component\+Disable} functions or with the N\+S\+\_\+\+L\+OG environment variable.

Log\+Component names should be simple string tokens, {\itshape i.\+e.}, \char`\"{}\+Arf\+Wifi\+Manager\char`\"{}, not \char`\"{}ns3\+::\+Arf\+Wifi\+Manager\char`\"{}.

This macro should be placed within namespace \hyperlink{namespacens3}{ns3}. If functions outside of namespace \hyperlink{namespacens3}{ns3} require access to logging, the preferred solution is to add the following \textquotesingle{}using\textquotesingle{} directive at file scope, outside of namespace \hyperlink{namespacens3}{ns3}, and after the inclusion of N\+S\+\_\+\+L\+O\+G\+\_\+\+C\+O\+M\+P\+O\+N\+E\+N\+T\+\_\+\+D\+E\+F\+I\+NE, such as follows\+: 
\begin{DoxyCode}
1 namespace ns3 \{
2   NS\_LOG\_COMPONENT\_DEFINE ("...");
3 
4   // Definitions within the ns3 namespace
5 
6 \} // namespace ns3
7 
8 using ns3::g\_log;
9 
10 // Further definitions outside of the ns3 namespace
\end{DoxyCode}
