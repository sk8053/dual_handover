\subsection{Detailed Description}
Interface for input file readers management. 

This interface perform the shared tasks among all possible input file readers. Each different file format is handled by its own topology reader.

Main topology reading function. 

The data is parsed and the results are returned in the passed lists. The rationale behind this choice is to allow non-\/progressive node I\+Ds in the topology files, as well as to separate the topology reader from the choices about actual IP number assignment and kind of links between nodes.

\begin{DoxyReturn}{Returns}
The container of the nodes created (or null if there was an error). 
\end{DoxyReturn}
