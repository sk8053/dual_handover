\subsection{Detailed Description}
A class which provides a relatively platform-\/independent Mutual Exclusion thread synchronization primitive. 

When more than one thread needs to access a shared resource (data structure or device), the system needs to provide a way to serialize access to the resource. An operating system will typically provide a Mutual Exclusion primitive to provide that capability. We provide plattorm-\/independent access to the O\+S-\/dependent capability with the \hyperlink{classns3_1_1SystemMutex}{System\+Mutex} class.

There are two operations\+: Lock and Unlock. Lock allows an executing \hyperlink{classns3_1_1SystemThread}{System\+Thread} to attempt to acquire ownership of the Mutual Exclusion object. If the \hyperlink{classns3_1_1SystemMutex}{System\+Mutex} object is not owned by another thread, then ownership is granted to the calling \hyperlink{classns3_1_1SystemThread}{System\+Thread} and Lock returns immediately, However, if the \hyperlink{classns3_1_1SystemMutex}{System\+Mutex} is already owned by another \hyperlink{classns3_1_1SystemThread}{System\+Thread}, the calling \hyperlink{classns3_1_1SystemThread}{System\+Thread} is blocked until the current owner releases the \hyperlink{classns3_1_1SystemMutex}{System\+Mutex} by calling Unlock.
