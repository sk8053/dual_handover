\subsection{Detailed Description}
Protocol header serialization and deserialization. 

Every Protocol header which needs to be inserted or removed from a \hyperlink{classns3_1_1Packet}{Packet} instance must derive from this base class and implement the pure virtual methods defined here.

Sample code which shows how to create a new type of \hyperlink{classns3_1_1Header}{Header}, and how to use it, is shown in the sample file samples/main-\/packet-\/header.\+cc

This method is used by \hyperlink{classns3_1_1Packet_a0961eccf975d75f902d40956c93ba63e}{Packet\+::\+Remove\+Header} to re-\/create a header from the byte buffer of a packet. The data read is expected to match bit-\/for-\/bit the representation of this header in real networks.

Note that data is not actually removed from the buffer to which the iterator points. Both \hyperlink{classns3_1_1Packet_a0961eccf975d75f902d40956c93ba63e}{Packet\+::\+Remove\+Header()} and \hyperlink{classns3_1_1Packet_aadc63487bea70945c418f4c3e9b81964}{Packet\+::\+Peek\+Header()} call \hyperlink{classns3_1_1Header_a78be9400bb66b2a8543606f395ef5396}{Deserialize()}, but only the Remove\+Header() has additional statements to remove the header bytes from the underlying buffer and associated metadata.
