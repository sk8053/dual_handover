\subsection{Detailed Description}
This \hyperlink{classns3_1_1SpectrumChannel}{Spectrum\+Channel} implementation can handle the presence of \hyperlink{classns3_1_1SpectrumPhy}{Spectrum\+Phy} instances which can use different spectrum models, i.\+e., different \hyperlink{classns3_1_1SpectrumModel}{Spectrum\+Model}.

\begin{DoxyNote}{Note}
It is allowed for a receiving \hyperlink{classns3_1_1SpectrumPhy}{Spectrum\+Phy} to switch to a different \hyperlink{classns3_1_1SpectrumModel}{Spectrum\+Model} during the simulation. The requirement for this to work is that, after the \hyperlink{classns3_1_1SpectrumPhy}{Spectrum\+Phy} switched its \hyperlink{classns3_1_1SpectrumModel}{Spectrum\+Model}, \hyperlink{classns3_1_1MultiModelSpectrumChannel_af365f7a37625876a9e476e6eb6376f7e}{Multi\+Model\+Spectrum\+Channel\+::\+Add\+Rx} () is called again passing the pointer to that \hyperlink{classns3_1_1SpectrumPhy}{Spectrum\+Phy}. 
\end{DoxyNote}

Note that a \hyperlink{classns3_1_1SpectrumPhy}{Spectrum\+Phy} that only transmits (without receiveing ever) does not need to be added to the channel.

This method is to be implemented by all classes inheriting from \hyperlink{classns3_1_1SpectrumChannel}{Spectrum\+Channel}.
